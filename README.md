# OOP_Lab7_Poryshkin
# Лабораторна робота №7 - Серіалізація/десеріалізація колекції об'єктів

## Опис проекту

Цей проект є продовженням лабораторної роботи №6 з додаванням функціоналу серіалізації та десеріалізації колекції об'єктів у формати CSV та JSON. Програма призначена для управління колекцією рослин з можливістю збереження та відновлення даних з файлів.

## Функціональність

### Основні можливості
- **Управління колекцією рослин**: додавання, перегляд, пошук, видалення
- **Серіалізація у CSV**: збереження колекції у текстовий формат з роздільниками
- **Серіалізація у JSON**: збереження колекції у JSON формат
- **Десеріалізація**: завантаження колекції з CSV та JSON файлів
- **Статичні методи**: робота з глобальними властивостями колекції
- **Валідація даних**: перевірка коректності введених даних

### Меню програми
1 - Додати рослину
2 - Переглянути всі рослини
3 - Знайти рослину
4 - Продемонструвати поведінку
5 - Видалити рослину
6 - Продемонструвати static-методи
7 - Зберегти колекцію об'єктів у файлі
8 - Зчитати колекцію об'єктів з файлу
9 - Очистити колекцію об'єктів
0 - Вийти з програми

Клас Plant
Основний клас, що представляє рослину з властивостями:

Тип (Tree, Shrub, Flower, Grass, Fern, Succulent, Vine)

Вік (0-5000 років)

Висота (0.01-115.7 метрів)

Дата посадки (після 1900 року, не в майбутньому)

Цвітуча (так/ні)

Клас PlantCollection
Менеджер колекції рослин з методами:

TryAddPlant() - додавання рослини

TryRemovePlant() - видалення за індексом

FindPlantsByType() - пошук за типом
